## What this project is

A desktop FBCA (Feature-Based Component Architecture) app generated by
`@bloomneo/bloom`. It runs as an Electron app with:

- **Frontend (renderer)**: React + `@bloomneo/uikit` — component library, themes, layouts
- **Backend (main process)**: Express + `@bloomneo/appkit` — auth, local SQLite DB, etc.
- **Architecture**: Features live in `src/web/features/` (renderer) and
  `src/desktop/main/features/` (main process). Each feature is self-contained.

## Key conventions

### Main process (AppKit)
- Always use `xxxClass.get()` — never `new SomethingClass()`
- Database is SQLite (better-sqlite3) — `DATABASE_URL=file:./app.db`
- Wrap async routes: `error.asyncRoute(async (req, res) => { ... })`
- Auth chain: `auth.requireLoginToken()` THEN `auth.requireUserRoles(['role.level'])`

### Renderer (UIKit)
- Wrap app in `<ThemeProvider>` — required for all UIKit components
- Renderer communicates with main via the local Express server (not IPC for API calls)
- Auto-routing: files in `features/*/pages/` become routes automatically

## Environment

Desktop apps use a `.env` file at project root:

```bash
DATABASE_URL=file:./app.db
```
